Frequently asked questions

Clear answers about what TOLZA is today, what it is not, and how the closed test works.

What is TOLZA?

TOLZA is a shopping-decision platform under development. Its goal is to help users search, build a basket, compare shopping choices and understand potential savings before deciding where to buy.

Is TOLZA live for public shopping?

No. The current Android release is an invitation-only closed test. This website is an information and compliance site, not a public shopping application.

Are the prices and savings real?

No. The current build uses synthetic demonstration products, prices and savings to test the journey and comparison logic.

Can I place an order or make a payment?

No. TOLZA does not currently provide checkout, payment, ordering or delivery.

Does TOLZA scrape retailer websites?

No. The current foundation has no live retailer APIs, real provider data or scraping.

Which languages are supported?

The current app foundation includes English, Gujarati, Hindi, Marathi, Tamil, Telugu and Malayalam. The production availability of each language may be controlled later.

Does the current app use GPS?

Production GPS integration is not part of the submitted Alpha. Future optional location assistance will be added only with clear permission and a manual pincode fallback.

Does TOLZA collect personal data?

The current submitted Alpha does not transmit Google Play required user data types off the device. Support emails contain only information the sender chooses to provide. See the Privacy Policy for details.

How can I delete my data?

Clear app data or uninstall the current Alpha to remove local test data. You can also use the in-app privacy route or the web-based instructions on the Account Deletion page.

When will TOLZA launch publicly?

No public launch date is promised. Production backend, real OTP, serviceability, operational controls, provider integrations, security and a new controlled release must be completed first.

How do I join the closed test?

Closed-test access is invitation-only. Approved testers must use the Google account included in the tester list and join through the official opt-in link after the release is approved.

How do I report a problem?

Use the Support page and include the device, Android version, screen, expected result and actual result without sending private credentials.
